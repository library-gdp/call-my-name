# 맞춤 자기소개서 작성

## 원칙

- 사용자 자료와 공개 출처에 근거하라. 없는 경험, 수치, 동기, 기업 정보를 만들지 마라.
- 나이, 성별, 사진, 가족관계 등 민감하거나 차별로 이어질 수 있는 정보는 사용자가 명시적으로 요구하거나 지원 양식상 필요한 경우에만 활용하라. 적합도 점수에는 반영하지 마라.
- 주민등록번호, 상세 주소, 계정 비밀번호 등 불필요한 개인정보를 요청하거나 저장하지 마라.
- 로그인, CAPTCHA, robots.txt 또는 사이트 이용약관을 우회하지 마라. 접근할 수 없는 공고는 회사 채용 페이지 등 공개 원문을 찾거나 사용자에게 URL/본문 제공을 요청하라.
- 현재 공고를 찾거나 회사 정보를 조사할 때는 반드시 웹을 검색하고, 공고 상태와 마감일을 확인하며, 최종 결과에 직접 링크와 확인 날짜를 포함하라.

## 작업 모드 선택

사용자의 요청에 필요한 단계부터 시작하라.

1. 자료만 주어졌으면 `프로필 구축`을 수행하라.
2. 프로필이 있고 공고 추천을 원하면 `공고 탐색`을 수행하라.
3. 특정 공고나 질문 문항이 있으면 `자기소개서 작성`을 수행하라.
4. 정보가 일부 부족해도 진행 가능한 부분을 먼저 완성하고, 작성에 결정적인 사실만 짧게 질문하라.

## 프로필 구축

이력서, 경력기술서, 성적표, 포트폴리오, 대화 내용을 읽고 다음을 구조화하라.

- 목표: 희망 직무, 산업, 지역, 고용형태, 경력 수준, 지원 제약
- 경력: 회사·역할·기간·책임·도구·성과·성과 수치와 근거
- 교육: 학교·전공·기간·학점·관련 과목·프로젝트
- 역량: 기술, 자격증, 언어, 도메인 지식, 협업 방식
- 증거 사례: 상황, 과제, 행동, 결과가 확인되는 에피소드
- 공백과 불확실성: 누락 정보, 서로 충돌하는 날짜나 수치, 확인 필요 주장

먼저 `지원자 프로필 요약`을 제시하고, 이어서 `확인할 사실`을 최대 5개 제시하라. 원자료에서 사실과 해석을 구분하고 각 핵심 주장에 출처 파일명이나 사용자 발언을 표시하라.

## 공고 탐색

웹 검색을 사용하여 사용자가 지정한 사이트와 회사 채용 페이지를 함께 조사하라. 검색 결과 요약만 믿지 말고 가능한 경우 공고 원문을 열어 확인하라.

각 공고에서 회사, 직무, 경력 요건, 필수·우대 역량, 근무지, 고용형태, 게시일, 마감일, 원문 URL을 수집하라. 중복 공고는 회사·직무·근무지 기준으로 합쳐라. 이미 마감됐거나 상태를 확인할 수 없는 공고는 명확히 표시하라.

적합도를 다음 기준으로 설명하라. 숫자 점수는 근거가 있을 때만 보조적으로 사용하라.

- 필수요건 충족
- 직무 역량 및 경험의 직접성
- 증명 가능한 성과와 우대요건
- 사용자의 직무·산업·지역 선호
- 부족한 요건과 지원 리스크

상위 공고를 비교표로 제시하고, 각 추천마다 `왜 적합한가`, `부족한 점`, `자소서에서 강조할 증거`를 포함하라. 사실 주장에는 출처 링크를 붙이고 확인 날짜를 적어라.

## 자기소개서 작성

공고 원문, 회사 공식 홈페이지·채용 페이지, 신뢰할 수 있는 최신 공개 자료를 조사하라. 회사의 가치나 사업 특성을 일반적인 칭찬으로 대체하지 말고, 지원자의 실제 경험과 연결되는 요소만 선택하라.

문항별 글자 수, 언어, 문체를 먼저 확인하라. 별도 문항이 없으면 다음 구조를 사용하라.

1. 직무와 연결되는 핵심 주장
2. 구체적인 상황·행동·결과와 수치
3. 해당 경험이 JD의 요구 역량을 입증하는 방식
4. 회사·조직의 구체적 특성과 입사 후 기여

초안 뒤에 다음을 함께 제공하라.

- `근거 매핑`: 문단별로 사용한 사용자 사실과 JD 요구사항
- `검증 필요`: 사용자가 확인해야 할 수치·명칭·날짜
- `사용하지 않은 핵심 소재`: 다른 문항에 활용할 수 있는 사례
- 글자 수(공백 포함 여부를 명시)

문항별 세부 템플릿과 품질 점검표가 필요할 때 [references/writing-guide.md](references/writing-guide.md)를 읽어라.

## 결과 품질 점검

- 모든 문장이 지원자 사실, 공고, 또는 출처 있는 기업 정보에 근거하는가?
- 다른 회사에도 그대로 붙일 수 있는 문장을 제거했는가?
- JD 키워드를 나열하지 않고 행동과 결과로 입증했는가?
- 같은 사례와 표현을 여러 문항에서 반복하지 않았는가?
- 제한 글자 수를 지켰는가?
- 채용공고 URL, 마감일, 정보 확인 날짜가 포함됐는가?
