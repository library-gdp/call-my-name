# 저장소 지침

## 공용 언어

이 프로젝트의 공용 언어는 한국어입니다. 문서, 이슈, 커밋 메시지, 코드 리뷰, 에이전트의 설명과 결과물은 원칙적으로 한국어로 작성합니다. 단, 코드 식별자, 파일명, 경로, 명령어, API 이름처럼 정확한 기술 표기가 필요한 항목은 원문을 유지합니다.

## 프로젝트 구조 및 모듈 구성

이 저장소는 Codex 플러그인 `call-my-name`을 개발하는 프로젝트입니다. 저장소 루트 전체가 플러그인은 아니며, 배포 단위는 `call-my-name/` 디렉터리로 한정합니다.

- `AGENTS.md`: 저장소에서 작업하는 에이전트가 따를 개발 지침입니다. 플러그인 배포물에 포함하지 않습니다.
- `.agents/`: 이 저장소와 플러그인을 개발·관리하기 위한 에이전트 자산과 스킬을 둡니다. 플러그인 배포물에 포함하지 않습니다.
- `.codex/`: 이 저장소에서만 사용하는 Codex 프로젝트 설정을 둡니다. 플러그인 배포물에 포함하지 않습니다.
- `call-my-name/`: 설치·배포 가능한 Codex 플러그인의 루트입니다.
  - `.codex-plugin/plugin.json`: 플러그인 매니페스트입니다.
  - `skills/`: 플러그인 사용자에게 제공할 스킬만 둡니다.
  - 각 스킬의 `skills/*/SKILL.md`는 메타데이터, 운영 규칙, 작업 흐름 및 출력 점검 기준을 정의합니다.
  - 각 스킬의 `agents/openai.yaml`은 해당 스킬의 사용자 표시 정보와 기본 프롬프트를 정의합니다.
  - 각 스킬의 `references/`는 필요할 때 불러오는 상세 지침을 포함합니다.

파일의 형식이 아니라 배포 목적을 기준으로 위치를 결정합니다. 플러그인을 개발하기 위한 Skill, MCP 설정, Agent 구성, 자동화 도구는 각각 `.agents/`, `.codex/` 등 저장소 루트의 개발 영역에 둡니다. 최종 사용자가 플러그인을 설치했을 때 직접 사용해야 하는 자산만 `call-my-name/` 아래에 둡니다. 같은 `SKILL.md`나 `agents/openai.yaml` 형식이라도 개발 보조용이면 플러그인 디렉터리에 넣지 않습니다.

플러그인 스킬의 핵심 지침은 `SKILL.md`에 간결하게 유지하고, 보충 설명은 `references/`에 배치한 뒤 상대 경로로 연결합니다. 새 스킬의 디렉터리 이름은 소문자 케밥 표기법을 사용합니다. 사용자용 스킬은 `call-my-name/skills/`에, 저장소 개발용 스킬은 `.agents/skills/`에 추가합니다.

변경 후에는 `call-my-name/`만으로 플러그인이 유효한지 검증하고, 개발 전용 파일이 그 안에 섞이지 않았는지 확인합니다. 관련 명령어로 스킬 파일 목록과 제목 구조를 검토하고, YAML 텍스트가 UTF-8로 올바르게 표시되는지 확인하며, 공백 오류를 탐지합니다. 제출하기 전에 Codex에서 대표적인 한국어 자기소개서 요청으로 플러그인 스킬을 실행하고 참조 파일이 올바르게 로드되는지 확인합니다.

## 보안 및 출처 무결성

이력서, 자격 증명, 상세 주소, 주민등록번호 또는 그 밖의 지원자 개인정보를 절대 커밋하지 않습니다. 공개 출처를 사용하고, 접근 제한을 준수하며, 채용 공고 상태와 마감일을 확인하고, 출처로 확인된 사실과 해석을 구분해야 한다는 스킬의 요구 사항을 유지합니다.
