# `call-my-name` 아키텍처

## 모듈

### User Analysis

사용자 컨텍스트를 수집하고 파일과 MCP 데이터를 파싱하여 구조화한다. 사용자의 경험에 근거해 역량을 분석하고 출처를 추적하며, 분석 결과에 대한 사용자의 검토와 승인을 반영한다.

### 모집공고 수집·분석 및 추천

허용된 방식으로 모집공고를 수집하여 저장하고 최신 상태로 갱신한다. 공고의 기업·직무·요건을 분석하고 사용자 컨텍스트와 비교하여 지원 적합도를 산정하고 공고를 추천한다.

### 지원서류 작성

공고에 적합한 사용자의 경험과 역량을 선정하고 사용자 승인을 받는다. 공고별 맞춤 지원서류를 생성하고 요구 형식으로 변환한 뒤 품질을 검사하여 최종 검토를 지원한다.

### 데이터 및 정책 관리

모듈이 공통으로 사용하는 데이터와 정책을 관리한다. 개인정보와 인증정보를 보호하고 데이터의 저장 수명주기, 출처, 승인 상태 및 모듈 간 데이터 계약을 일관되게 유지한다.

## Skill 목록

| Module | Skill Name | Description (When to use) |
| --- | --- | --- |
| User Analysis | `collect-user-context` | 지원자 분석에 필요한 인적·경력·학력·자격 정보를 파일, MCP 데이터소스 및 사용자 응답으로 수집하고 구조화할 때 사용한다. 이력서, 자기소개서, 포트폴리오 등 PDF·DOCX·PPTX 형식의 문서에서는 경험, 성과, 기술과 출처 정보를 추출한다. 자료가 부족하면 추가 질문으로 보완하고, 추출이 불확실한 내용은 사용자 확인 대상으로 표시한다. |
| User Analysis | `analyze-user-capabilities` | 수집된 경험을 근거로 사용자의 기술과 직무 역량을 정의하고, 각 역량을 관련 경험 및 문서 출처와 연결할 때 사용한다. 근거의 강도와 숙련도를 평가하되 추론과 확인된 사실을 구분한다. |
| User Analysis | `review-user-profile` | 구조화된 사용자 정보와 역량 분석 결과를 검토 가능한 프로필로 제시하고 사용자의 정정·승인을 반영할 때 사용한다. 승인된 프로필은 공고 추천과 지원서류 작성에 재사용한다. |
