# call-my-name 기능명세서 초안

- 사용자 컨텍스트 수집
  - 사용자의 지원 적합도와 지원 서류 작성에 필요한 정보를 수집한다.
  - 기본 정보는 이름, 생년월일, 성별을 포함한다. 생년월일과 성별은 채용 공정성 및 필요성을 검토한 뒤, 필요한 경우에만 수집한다.
  - 경력 정보는 회사명, 직무, 근속 기간을 포함한다.
  - 학력 정보는 학교명, 학위, 전공, 입학일 및 졸업일을 포함한다.
  - 자격 정보는 자격증명, 발급 기관, 취득일, 유효기간, 자격증 번호를 포함한다. 자격증 번호는 검증에 꼭 필요한 경우에만 수집하며 결과물에 노출하지 않는다.
  - 기존 자기소개서, 이력서, 포트폴리오에서 경험과 역량을 추출한다.
  - 사용자가 지정한 경로에 파일을 제공하면 해당 파일을 입력으로 사용한다.
  - 파일만으로 부족한 정보는 사용자에게 직접 질문하여 보완한다.
  - 사용자가 연결을 허용한 Notion 등 MCP 데이터소스에서 컨텍스트를 불러온다.
  - PDF, DOCX, PPTX 등 비평문 형식의 지원 서류는 적절한 파서로 텍스트와 구조화 가능한 정보를 추출한다.
  - 지원 서류 파일 형식별 파싱 가능 여부, 표·이미지·스캔 문서의 추출 품질, 지원 라이브러리의 라이선스와 실행 환경을 검증한다.
  - 수집한 개인정보와 원본 파일은 작업 목적에 필요한 범위에서만 사용하고, 저장 위치·보존 기간·삭제 방법을 사용자에게 알린다.

- 사용자 컨텍스트 분석
  - 수집한 사용자 정보를 바탕으로 경험, 기술, 직무 역량, 자격을 목록으로 구조화한다.
  - 각 역량에 대해 근거가 된 경험 또는 문서 출처와 숙련도 평가를 함께 기록한다.
  - 숙련도 평가는 사용자 경력의 기간, 역할, 산출물, 반복 경험 등 확인 가능한 근거를 바탕으로 작성하고, 불확실한 평가는 사용자 검토 대상으로 표시한다.
  - 분석 결과를 사용자가 검토할 수 있는 Markdown 보고서로 작성한다.
  - 사용자의 정정·승인 내용을 반영하여 확정된 컨텍스트를 이후 공고 추천과 지원 서류 작성에 재사용한다.
  - 역량 분류 체계, 숙련도 기준, 사용자 승인 전 분석 결과의 사용 범위를 검증한다.

- 모집공고 수집
  - 사용자가 지정한 채용 사이트 또는 모집공고 링크에서 모집공고를 수집한다.
  - 사람인, LinkedIn, 잡코리아, 자소설닷컴 등 데이터소스별 접근 가능 여부와 이용약관, robots.txt, API 제공 여부를 확인한 뒤 허용된 방식으로 수집한다.
  - 수집 시 대상 사이트에 과도한 부하를 주지 않도록 요청 빈도 제한, 재시도 정책, 중복 요청 방지 정책을 적용한다.
  - 처음 사용하는 데이터소스는 페이지 구조와 접근 제약을 분석한 뒤 재사용 가능한 수집 방식을 구현한다.
  - 로그인이 필요한 경우 사용자에게 해당 사이트에서 직접 인증하도록 안내하거나, 사용자가 명시적으로 허용한 세션에서만 인증을 수행한다.
  - 아이디, 비밀번호, MFA 코드, 세션 ID, Bearer Token 등 인증 정보는 저장·기록·공유하지 않는다.
  - 모집공고마다 제목, 모집 기업, 고용형태, 직무, Job Description, 채용 조건, 원문 URL, 수집 시각, 마감일 또는 공고 상태를 저장한다.
  - 공고 식별자 또는 원문 URL을 기준으로 최신 정보를 갱신하고 중복 수집을 방지한다.
  - 사이트별 수집 안정성, 로그인·자동화 허용 범위, 데이터 보존 방식, 공고 상태 및 마감일 확인 방법을 검증한다.

- 모집공고 분석 및 추천
  - 수집한 Job Description과 채용 조건을 분석하여 직무, 필수·우대 역량, 경력 수준, 고용 조건을 구조화한다.
  - 구조화된 공고 정보와 확정된 사용자 컨텍스트를 비교해 지원 적합도를 산정한다.
  - 적합한 공고만 추천하고, 추천 근거와 부족하거나 확인이 필요한 요건을 함께 제시한다.
  - 공고 원문에서 확인된 사실과 모델의 해석·추론을 구분해 표시한다.
  - 적합도 산정 기준, 직무별 가중치, 추천 결과의 편향 및 사용자 피드백 반영 방식을 검증한다.

- 지원 서류 작성
  - 사용자가 선택한 모집공고별로 자기소개서, 이력서, 포트폴리오 초안을 작성한다.
  - 각 서류에는 공고와 가장 관련성이 높은 경험과 역량을 우선 반영하며, 관련 없는 이력이나 검증되지 않은 성과는 포함하지 않는다.
  - 작성 전에 반영할 역량과 경험을 사용자에게 제시하고, 사용자 피드백과 승인을 반영한다.
  - 자기소개서는 Markdown 형식으로 작성한다.
  - 이력서와 포트폴리오는 지정된 예시 템플릿을 기반으로 PDF, DOCX, PPTX 등 요구 형식으로 작성한다.
  - 템플릿 파일의 제공 위치, 사용 권한, 형식별 생성·변환 품질, 채용 시스템 호환성을 검증한다.
  - 생성된 서류는 제출 전 사실관계, 개인정보 노출, 공고 요구사항 충족 여부를 사용자가 검토할 수 있도록 한다.
